First Impressions of Flask

Slide 4

‘Lightweight’ positioning: Flask is not a comprehensive framework, providing only core functionality with extensions for additional features.
‘Microframework’: Simple core that can be extended to enhance functionality.
‘Dual-layer standard’: Adheres to WSGI and HTTP protocols, forming the foundation of Python web development.

Slide 5

Line-by-Line Code Analysis:
· from flask import Flask: Imports the Flask class
· app = Flask(__name__): Creates an application instance
· @app.route(“/”): URL mapping to a function
· def index(): return “Hello, Flask!”: View function returning a response
· app.run(): Launches development server
· Access 127.0.0.1:5000: Localhost IP and default port

Slide 6

Core Components

Slide 7

Routing Introduction:
· URL mapping table locates handler functions
· @app.route decorates functions as web endpoints
· Routing = Controller layer in MVC

Static vs Dynamic Routing:
· Static: fixed content
· Dynamic: URL variables (<username>, <int:user_id>) for personalized responses

Slide 8

Jinja2 Template Engine:
· Avoids HTML concatenation in code
· {{ '{{ ... }}' }} outputs escaped content
· {% raw %}{% ... %}{% endraw %} for logical control
· Template inheritance enables base templates

render_template() Workflow:
1. Search templates folder
2. Create template context with Python variables
3. Execute Jinja2 to generate HTML
4. Return HTTP response
· Pass multiple variables: render_template(“page.html”, name=name, age=age, items=items)

Slide 9

Request object - Retrieving request data:
· GET: retrieve data, visible URL parameters
· POST: submit data, hidden in body
· request.args, request.form: MultiDict
· request.json: requires Content-Type header
· Ajax: update page without refresh

Slide 10

Redirection and Session Persistence:
· HTTP status code 302 Found instructs browser to redirect
· Post/Redirect/Get prevents duplicate submissions
· Session mechanism:
   · Server stores session data
   · Session ID in cookie
   · Browser sends cookie on subsequent requests
   · Server locates session data via ID

· app.secret_key: signs session cookie, prevents tampering
· Use session dictionaries for state persistence
· url_for(): generate URLs dynamically

User Login Process:
1. Submit login form (POST)
2. Validate credentials
3. Record login in Session
4. Redirect to backend page
· Login state maintained until browser closes

Deployment and Scaling Overview

Slide 22

Production Environment Deployment:
· Development vs Production: debugging convenience vs performance, stability, security
· WSGI Servers: concurrent request handling, process management, performance enhancement

WSGI Server Selection:
   · Waitress: pure Python, cross-platform, simple
   · Gunicorn: Unix pre-fork, high performance
   · app:app → module name + Flask instance
   · Binding 0.0.0.0 → accessible from all IPs

Slide 23

Security Configuration:
· Debug mode risks: exposes source code and variables
· SECRET_KEY best practices:
   · Generate via os.urandom(24)
   · Configure via environment variables
   · Use distinct keys for production & development
